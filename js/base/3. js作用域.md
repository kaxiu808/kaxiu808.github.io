### 作用域

作用域是指程序中定义变量的可访问性和可见性的范围，它决定了在代码中的哪些位置可以访问或引用特定变量，函数或对象。

在 JavaScript 中，作用域分为全局作用域和局部作用域。全局作用域是指在代码中任何地方都可以访问到的变量或函数的范围，而局部作用域是指在函数内部定义的变量或函数的范围，只能在函数内部访问。

JavaScript，在不同的作于域中定义的变量具有不同的可见性和生命周期。常见的作于域类型有以下几种：

- 1.全局作于域(Global) :全局作于域是在整个程序中都可访问的作于域。在全局作于域中变量可以在程序的任何位置被访问。
- 2.函数作于域(Function) :函数作于域是在函数内部定义的作于域。在函数作用域中声明的变量只能在函数内部被访问，它们对于函数外部是不可见的。每当函数被调用时，都会创建一个新的函数作于域。
- 3.块作于域(Block) :块作于域是在代码块(通常由花括号 {} 包围) 内定义的作于域。在块级作于域中声明的变量只能在该块内部被访问，对于外部是不可见的，而在块外部是不可见的。在ES6中引入的let和const关键字可以创建块级作于域。

```js
var a = 100
if (2>1){
    var b = 200
}
console.log(b)
```
>在python和js的能开辟作于域的只有函数，在js中，函数的作于域是函数被调用时创建的，函数调用结束后，作于域被销毁。


### 匿名函数

```js
// 匿名函数
var foo = function(){
    console.log("foo...")
}
foo()

```
> 匿名函数 ,没有其它功能，就是为了解决全局变量污染，而单独为变量(函数)开辟的作于域

```js
// 匿名函数自制行
(function (x,y){
    console.log(x+y)
})(1,2)

!(function (x,y){
    console.log(x+y)
})(1,3)
```
## 闭包

>闭包是javastipt语言的一个难点，也就是它的特色，很多高级应用都是依靠必闭包实现的。
闭包与变量的作用域以及变量的生命周期密切相关。
在计算机科学中，闭包(Closure),又称词法闭包(Lexical Closure)或函数闭包(function closures)，是引用了自由变量(外部非全局)函数。

简单来说就是一个函数定义中引用了函数外定义的变量，并且该函数可以在其定义环境被执行
